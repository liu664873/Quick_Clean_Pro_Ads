# QuickCleanPRO 马甲包复制改包清单

这份清单用于从母包复制出新马甲包时逐项核对。原则是优先改配置和资源，尽量不改业务代码。

## 1. 基础包信息

- `applicationId`：检查 `app/build.gradle.kts` 中的 `defaultConfig.applicationId`。
- `namespace`：通常保持与代码包一致；如确实改包结构，再同步调整。
- `versionCode/versionName`：按发版节奏更新。
- 应用名称：替换 `res/values/strings.xml` 中的 `app_name`。
- 应用图标：替换 `mipmap-*` 和必要的 launcher/round icon 资源。
- 主题 key：检查 `app_profile_key`、`app_theme_key` 相关资源，确保 `AppProfileLoader` 能读到正确品牌配置。

## 2. 法务与品牌配置

- 隐私政策 URL：更新 `config/app.defaults.properties` 或本地覆盖中的 `ADV_PRIVACY_URL`，同时检查 `terms_of_service_url/privacy_policy_url` 字符串资源。
- 服务协议 URL：更新 `ADV_TERMS_URL` 和对应字符串资源。
- 品牌文案：全局搜索旧包名、旧应用名、旧公司名、旧隐私链接。
- 主题颜色/图片：优先改资源和 theme，不在页面里写品牌分支。

## 3. 广告与归因配置

- AdMob：更新 `ADV_ADMOB_APP_ID`、Banner、Interstitial、Native、Open、Rewarded ID。
- 广告策略：检查 `res/raw/ad_policy.json`、`res/raw/adload_config.json`、`res/raw/native_ad_policy.json`、`res/raw/native_ad_ids.json`。
- `areaKey`：保持代码侧 `AdAreaKeys` 与产品表、raw policy 一致。
- ThinkingData：更新 `ADV_THINKING_APP_KEY`、`ADV_THINKING_SERVER_URL`。
- Singular：更新 `ADV_SINGULAR_API_KEY`、`ADV_SINGULAR_SECRET`。
- Firebase：替换 `google-services.json`，确认包名匹配。
- Facebook/TikTok/Pangle 等平台：按投放要求更新对应 app id/token。

## 4. 安全与第三方服务

- Trustlook：更新 `TRUSTLOOK_QUICKCLEANPRO_API_KEY` 或新包对应 key。
- Play Integrity：更新 `ADV_PLAY_INTEGRITY_CLOUD_PROJECT_NUMBER`、`ADV_PLAY_INTEGRITY_PARSE_TOKEN_KEY`。
- 签名校验：更新 `ADV_SAFE_EXPECTED_SIGNATURES`。
- 发布签名：确认 release signing 配置，不要误用 debug 签名发正式包。

## 5. 通知与入口

- 持久通知快捷入口：检查 `quickCleanProNotificationProfile()` 中的 route、requestCode、icon。
- Push 配置：检查 `res/raw/push.json` 和 SDK 通知默认配置。
- 通知场景 route：检查 `NotificationRouteAliases`、`ToolNotificationIntentFactory`。
- 通知图标：替换 small icon、persistent shortcut icon。

## 6. 功能与路由

- 首页功能：检查 `FeatureCatalog`、Home tabs、工具箱入口是否符合新包需求。
- 路由映射：新增/删除功能时同步 `FeatureCatalog`、路由注册、广告位映射。
- 文件管理详情页：使用统一 route helper，不直接拼裸字符串。
- 权限入口：确认文件权限、Usage Access、Location、Notification Listener、Overlay、通知权限是否仍需要。

## 7. 埋点与广告验收

- 冷启动、引导页、首页、核心功能点击、权限弹窗、协议跳转按埋点表验收。
- 进入功能广告：确认日志出现 `main*Adv`。
- 返回功能广告：确认日志出现 `returnFrom*Adv`。
- 完成广告：确认清理/删除/测速完成使用对应 finish 广告位。
- 广告生命周期由 SDK 自动上报，业务层不要重复报 `ad_impression/ad_click/ad_close`。

## 8. 复制后必查

- 全局搜索旧应用名、旧包名、旧域名、旧 app id。
- 跑广告配置静态测试，确认 raw policy 和 `AdAreaKeys` 一致。
- 手动走冷启动、首页进入功能、权限拒绝、功能返回、通知点击主链路。
- 不确认业务行为时，不要为了马甲包单独改页面逻辑；优先通过配置、资源、FeatureCatalog 处理。
